# مهارة OpenCode: منع هلوسات الذكاء الاصطناعي

> مهارة وكيل OpenCode تمنع هلوسات الذكاء الاصطناعي من خلال فرض التحقق والاعتراف بعدم اليقين والردود القائمة على الأدلة.

[English](./README.md) | [中文](./README_CN.md) | [Русский](./README_RU.md) | [Français](./README_FR.md) | [Deutsch](./README_DE.md) | [Español](./README_ES.md) | [Português](./README_PT.md) | العربية

## الميزات

- **تحقق قبل أن تؤكد** — اقرأ الملفات، نفّذ التعليمات البرمجية، استشر الوثائق قبل الاستنتاج
- **اعترف بعدم اليقين** — ميّز بوضوح بين التخمين والحقائق المؤكدة
- **لا تختلق** — لا تصنع أبداً أخطاءً، APIs، نتائج اختبارات أو إصلاحات
- **استند إلى الأدلة** — استشهد بمسارات الملفات وأرقام الأسطر والمخرجات الفعلية
- **التحقق الذاتي** — قائمة تحقق من 10 نقاط قبل كل إجابة
- **حراسات توليد التعليمات البرمجية** — تحقق مزدوج قبل وبعد كتابة التعليمات البرمجية
- **قواعد لكل أداة** — قواعد محددة لمنع الهلوسة لأدوات read، write، edit، bash، grep، glob وغيرها
- **8 أنواع من الهلوسات** — لكل نوع استراتيجية الوقاية الخاصة به

## التثبيت السريع

### الطريقة 1: استنساخ المستودع (موصى به)

```bash
git clone https://github.com/magiccloudflash/opencode-skill-hallucination-prevention.git
```

ثم انسخ المهارة إلى إعدادات OpenCode الشاملة:

```bash
# macOS / Linux
cp -r hallucination-prevention ~/.config/opencode/skills/

# Windows PowerShell
Copy-Item -Recurse hallucination-prevention "$env:USERPROFILE\.config\opencode\skills\"
```

### الطريقة 2: التثبيت اليدوي

```bash
# macOS / Linux
mkdir -p ~/.config/opencode/skills/hallucination-prevention
cp hallucination-prevention/SKILL.md ~/.config/opencode/skills/hallucination-prevention/

# Windows PowerShell
mkdir -p "$env:USERPROFILE\.config\opencode\skills\hallucination-prevention"
Copy-Item hallucination-prevention/SKILL.md "$env:USERPROFILE\.config\opencode\skills\hallucination-prevention\"
```

### الطريقة 3: كقاعدة شاملة (تحميل تلقائي في كل جلسة — موصى به)

```bash
# macOS / Linux
cp AGENTS.md ~/.config/opencode/AGENTS.md

# Windows PowerShell
Copy-Item AGENTS.md "$env:USERPROFILE\.config\opencode\AGENTS.md"
```

> الملف الشامل `AGENTS.md` يُحمّل تلقائياً في كل جلسة، دون الحاجة لاستدعاء المهارة يدوياً.

## الاستخدام

### عند الطلب (الطريقة 1، 2)

يكتشف OpenCode المهارة تلقائياً. يقوم الوكيل بتحميلها عند الحاجة:

```javascript
skill({ name: "hallucination-prevention" })
```

### التحميل التلقائي (الطريقة 3)

تُحقن قواعد `AGENTS.md` تلقائياً في سياق كل جلسة. **لا حاجة لأي إجراء يدوي.**

## تكوين الأذونات

في ملف `opencode.json`:

```json
{
  "permission": {
    "skill": {
      "hallucination-prevention": "allow"
    }
  }
}
```

لوكلاء محددين:

```json
{
  "agent": {
    "plan": {
      "permission": {
        "skill": {
          "hallucination-prevention": "allow"
        }
      }
    }
  }
}
```

تدعم الأذونات أنماط البدل:

| الإذن | السلوك |
|-------|--------|
| `allow` | تُحمّل المهارة فوراً |
| `deny` | تُخفى المهارة عن الوكيل، يُرفض الوصول |
| `ask` | طلب تأكيد قبل التحميل |

## محتوى المهارة v2.0

### تصنيف 8 أنواع من الهلوسات

| النوع | الوصف | مثال | الوقاية |
|-------|-------|------|---------|
| **أ: واقعي** | تأكيد حقائق غير موثقة | "هذه الدالة تُرجع Promise" (دون قراءة الكود) | اقرأ المصادر، نفّذ الكود |
| **ب: هيكلي** | اختلاق ملفات/مسارات | "المعالج موجود في src/routes/auth.ts" (الملف غير موجود) | استخدم glob/ls للتحقق |
| **ج: سلوكي** | افتراض سلوك الكود | "الخطأ تم إصلاحه" (دون تشغيل المشروع) | نفّذ الكود وأظهر النتيجة |
| **د: خارجي** | اختلاق APIs/دوال | "Lodash لديه دالة groupByAndSort()" (غير موجودة) | تحقق من الوثائق أو المصادر |
| **هـ: نتائجي** | تخيل نتائج الاختبارات | "جميع الاختبارات ناجحة" (دون تشغيلها) | نفّذ الاختبارات وأظهر النتيجة الفعلية |
| **و: زمني** | إصدارات/تواريخ خاطئة | "Node v24 أضافت هذه الميزة" (v24 غير موجود) | تحقق من الإصدار وسجل التغييرات |
| **ز: سلطوي** | التظاهر بالتحقق | "لقد أكدت أن التغيير صحيح" (دون إعادة القراءة) | نفّذ التحقق وأظهر الأدلة |
| **ح: مرجعي** | اختلاق URLs/IDs/commits | "انظر https://github.com/foo/issues/99999" (رابط مزيف) | تحقق من URL أو أشر "لم يتم التحقق" |

### قواعد كل أداة

| الأداة | القاعدة الرئيسية |
|--------|-----------------|
| **read** | لا تخمن المحتوى؛ تحقق من وجود الملف؛ أعد القراءة بعد التحرير |
| **write** | أعد قراءة ما كتبته فوراً؛ لا تنشئ وثائق دون طلب |
| **edit** | اقرأ دائماً قبل التحرير؛ أعد القراءة بعد ذلك؛ احترم المسافات البادئة |
| **bash** | لا تختلق مخرجات الأوامر؛ أعد التشغيل للتحقق من الإصلاحات؛ اسأل قبل الإجراءات التدميرية |
| **grep** | تأكد من وجود الدوال/الفئات عبر grep |
| **glob** | استخدم glob للتحقق من الهيكل قبل الإشارة إلى المسارات |
| **question** | عند غموض التعليمات، اسأل أولاً، ثم تصرّف |

### قائمة التحقق من 10 نقاط

قبل كل إجابة، تحقق:

| # | السؤال | الإجراء إذا "لا" |
|---|--------|-----------------|
| 1 | هل قرأت بالفعل الملفات التي أتحدث عنها؟ | اقرأها الآن |
| 2 | هل نفّذت بالفعل الأوامر التي أبلغ عن مخرجاتها؟ | نفّذها الآن |
| 3 | هل تحققت بالفعل من الحزمة/API التي أشير إليها؟ | تحقق أو أشر إلى عدم اليقين |
| 4 | هل أنا متأكد أن رسالة الخطأ هذه حقيقية؟ | أظهر المخرجات الفعلية |
| 5 | هل أعدت قراءة الملفات بعد تحريرها؟ | أعد قراءتها الآن |
| 6 | هل أخمن هيكل الدليل؟ | اعرض قائمة الدليل |
| 7 | هل أخمن توقيع دالة؟ | اقرأ الكود المصدري |
| 8 | هل ادّعيت أن الإصلاح يعمل دون اختباره؟ | نفّذ الكود |
| 9 | هل أختلق خيارات الأوامر؟ | تحقق من --help |
| 10 | هل قد تحتوي إجابتي على بيانات مختلقة؟ | احتفظ فقط بالمخرجات الحقيقية |

### بروتوكول تثبيت السياق

1. **قبل العمليات الهامة على الملفات**: أعد قراءة ملفات المصدر ذات الصلة
2. **بعد 5+ استدعاءات للأدوات**: أعد قراءة الملفات المعدّلة، تحقق من الحالة المتراكمة
3. **عند العودة إلى مهمة**: أعد قراءة جميع الملفات ذات الصلة — لا تعتمد على الذاكرة

### قواعد خاصة بلغات/أطر العمل

- **JS/TS**: تحقق من package.json قبل الاستيراد؛ تحقق من tsconfig.json
- **Python**: تحقق من requirements.txt/pyproject.toml؛ تحقق من python --version
- **Rust**: تحقق من Cargo.toml؛ تحقق من rust-toolchain.toml
- **Go**: تحقق من go.mod؛ تحقق من go version
- **Shell**: تحقق من توفر الأوامر قبل كتابة النصوص البرمجية؛ اختبر الأوامر قبل اقتراحها

### قواعد خاصة بـ Git

- لا تختلق أبداً تجزئات commit — استخدم `git log`
- لا تفترض أبداً رسائل commit — استخدم `git log --oneline`
- لا تختلق أبداً أرقام PR أو IDs Issues
- بعد commit، نفّذ `git status` و `git log -1`
- قبل الادعاء بوجود فرع، نفّذ `git branch -a`

### مستويات الثقة

- **تم التحقق**: قرأت/نفّذت/تحققت مباشرة
- **محتمل**: أدلة قوية لكن دون تحقق مباشر
- **تخميني**: استدلال بناءً على أنماط مشابهة، يحتاج تحقق
- **غير معروف**: لا يمكن التحديد من المعلومات المتاحة

افتراضياً، استخدم مستوى **تم التحقق**.

### بروتوكول التصعيد (متى تتوقف وتسأل المستخدم)

توقف فوراً واسأل المستخدم عندما:
1. التعليمات غامضة، مع احتمالية تفسيرات متعددة
2. الأداة/المكتبة المطلوبة غير متوفرة
3. العملية تحتاج صلاحيات مرتفعة
4. ما تلاحظه يتعارض مع وصف المستخدم
5. المهمة تتطلب معرفة لا يمكنك التحقق منها
6. العملية قد تسبب فقدان بيانات أو مشاكل أمنية
7. تغير السياق كثيراً لمتابعته بدقة

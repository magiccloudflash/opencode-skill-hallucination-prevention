# OpenCode مہارت: AI فریب کی روک تھام

> ایک OpenCode ایجنٹ مہارت جو تصدیق، غیر یقینی کے اعتراف، اور ثبوت پر مبنی جوابات کو نافذ کرکے AI فریب کو روکتی ہے۔

## خصوصیات

- **بیان کرنے سے پہلے تصدیق کریں** — دعویٰ کرنے سے پہلے فائلیں پڑھیں، کوڈ چلائیں، دستاویزات دیکھیں
- **غیر یقینی کا اعتراف کریں** — قیاس آرائی اور تصدیق شدہ حقائق میں واضح فرق کریں
- **من گھڑت نہ بنائیں** — کبھی بھی غلطیاں، APIs، ٹیسٹ کے نتائج، یا اصلاحات ایجاد نہ کریں
- **ثبوت پر مبنی ہوں** — فائل کے راستے، لائن نمبر، اور حقیقی آؤٹ پٹ کا حوالہ دیں
- **خود جانچ کا طریقہ کار** — ہر جواب سے پہلے 10 نکاتی جانچ کی فہرست
- **کوڈ جنریشن کے تحفظات** — تحریر سے پہلے اور بعد میں تصدیق
- **خرابی سے نمٹنے کا پروٹوکول** — حقیقی خرابیاں پڑھیں، لفظ بہ لفظ دکھائیں، اصلاحات کی تصدیق کریں
- **8 اقسام کے فریب** — ہر قسم کے لیے مخصوص روک تھام کی حکمت عملی
- **ٹول کے مخصوص اصول** — read، write، edit، bash، grep، glob اور دیگر کے لیے الگ اصول

## تنصیب

### طریقہ 1: اس ریپو کو کلون کریں

```bash
git clone https://github.com/<your-org>/opencode-skill-hallucination-prevention.git
```

پھر مہارت کو اپنی عالمی opencode ترتیب میں کاپی کریں:

```bash
# macOS / Linux
cp -r hallucination-prevention ~/.config/opencode/skills/

# Windows PowerShell
Copy-Item -Recurse hallucination-prevention "$env:USERPROFILE\.config\opencode\skills\"
```

### طریقہ 2: دستی تنصیب

مہارت کی ڈائرکٹری بنائیں اور `SKILL.md` کاپی کریں:

```bash
# macOS / Linux
mkdir -p ~/.config/opencode/skills/hallucination-prevention

# Windows PowerShell
mkdir -p "$env:USERPROFILE\.config\opencode\skills\hallucination-prevention"
```

پھر [`hallucination-prevention/SKILL.md`](hallucination-prevention/SKILL.md) کو اس ڈائرکٹری میں رکھیں۔

### طریقہ 3: عالمی اصول کے طور پر (ہر سیشن میں خودکار لوڈ)

بہتر اصولوں کو اپنی عالمی AGENTS.md میں کاپی کریں:

```bash
# macOS / Linux
cat hallucination-prevention/SKILL.md >> ~/.config/opencode/AGENTS.md

# Windows PowerShell
Add-Content -Path "$env:USERPROFILE\.config\opencode\AGENTS.md" -Value (Get-Content "hallucination-prevention\SKILL.md")
```

یا اس ریپو میں فراہم کردہ علیحدہ [`AGENTS.md`](AGENTS.md) استعمال کریں۔

## استعمال

یہ مہارت OpenCode کے ذریعے خود بخود دریافت ہو جاتی ہے۔ ایجنٹ اسے دستیاب مہارتوں میں دیکھے گا اور جب مناسب ہوگا لوڈ کرے گا۔

اسے واضح طور پر لوڈ کرنے کے لیے:

```
skill({ name: "hallucination-prevention" })
```

### اجازت کی ترتیب

اپنی `opencode.json` میں، آپ رسائی کو کنٹرول کر سکتے ہیں:

```json
{
  "permission": {
    "skill": {
      "hallucination-prevention": "allow"
    }
  }
}
```

فی ایجنٹ اوور رائڈ کے لیے:

```json
{
  "agent": {
    "plan": {
      "permission": {
        "skill": {
          "hallucination-prevention": "allow"
        }
      }
    }
  }
}
```

## مہارت کا مواد v2.0

### فریب کی 8 اقسام

| قسم | تفصیل | مثال | روک تھام |
|------|--------|------|---------|
| **A: حقائقی** | غیر تصدیق شدہ حقائق بتانا | "یہ فنکشن Promise لوٹاتا ہے" (کوڈ پڑھے بغیر) | ماخذ پڑھیں، کوڈ چلائیں |
| **B: ساختی** | فائلوں/راستوں کا ایجاد کرنا | "ہینڈلر src/routes/auth.ts میں ہے" (فائل موجود نہیں) | glob/ls سے تصدیق کریں |
| **C: رویاتی** | کوڈ کے رویے کا قیاس کرنا | "خرابی اب ٹھیک ہو گئی" (پراجیکٹ چلائے بغیر) | کوڈ چلائیں اور نتیجہ دکھائیں |
| **D: خارجی** | APIs/فنکشنز کا ایجاد کرنا | "Lodash میں groupByAndSort() ہے" (موجود نہیں) | دستاویزات/ماخذ چیک کریں |
| **E: نتائجی** | ٹیسٹ کے نتائج کا تصور کرنا | "تمام ٹیسٹ پاس ہو گئے" (چلائے بغیر) | ٹیسٹ چلائیں، حقیقی آؤٹ پٹ دکھائیں |
| **F: زمانی** | غلط ورژن/تاریخ | "Node v24 نے یہ فیچر شامل کیا" (v24 موجود نہیں) | ورژن، تبدیل شدہ لاگ چیک کریں |
| **G: اختلافی** | جانچ کا دعویٰ کرنا | "میں نے تصدیق کر لی ہے" (دوبارہ پڑھے بغیر) | اصل میں جانچ کریں |
| **H: حوالاتی** | URLs/IDs/commits کا ایجاد کرنا | "https://github.com/foo/issues/99999 دیکھیں" (جعلی لنک) | URL کی تصدیق کریں یا "تصدیق نہیں ہوئی" کہیں |

### ٹول کے مخصوص اصول

| ٹول | بنیادی اصول |
|------|-------------|
| **read** | مواد کا اندازہ نہ لگائیں؛ فائل کے وجود کی تصدیق کریں؛ ترمیم کے بعد دوبارہ پڑھیں |
| **write** | لکھنے کے فوراً بعد دوبارہ پڑھیں؛ بغیر پوچھے دستاویزات نہ بنائیں |
| **edit** | ترمیم سے پہلے ہمیشہ پڑھیں؛ ترمیم کے بعد دوبارہ پڑھیں؛ انڈینٹیشن کا احترام کریں |
| **bash** | کمانڈ آؤٹ پٹ کا اندازہ نہ لگائیں؛ اصلاحات کی تصدیق کے لیے دوبارہ چلائیں؛ تباہ کن اقدامات سے پہلے پوچھیں |
| **grep** | فنکشنز/کلاسز کے وجود کی تصدیق کے لیے grep استعمال کریں |
| **glob** | راستوں کا حوالہ دینے سے پہلے ڈھانچے کی تصدیق کے لیے glob استعمال کریں |
| **question** | جب ہدایات مبہم ہوں، پہلے پوچھیں، پھر عمل کریں |

### سیاق و سباق کو مستحکم کرنے کا پروٹوکول

1. **فائل آپریشنز سے پہلے**: متعلقہ ماخذ فائلیں دوبارہ پڑھیں
2. **5+ ٹول کالز کے بعد**: ترمیم شدہ فائلیں دوبارہ پڑھیں، جمع شدہ حالت کی تصدیق کریں
3. **کسی کام پر واپسی پر**: تمام متعلقہ فائلیں دوبارہ پڑھیں — یادداشت پر بھروسہ نہ کریں

### زبان/فریم ورک کے اصول

- **JS/TS**: درآمد کرنے سے پہلے package.json چیک کریں؛ tsconfig.json چیک کریں
- **Python**: requirements.txt/pyproject.toml چیک کریں؛ python --version چیک کریں
- **Rust**: Cargo.toml چیک کریں؛ rust-toolchain.toml چیک کریں
- **Go**: go.mod چیک کریں؛ go version چیک کریں
- **Shell**: اسکرپٹ لکھنے سے پہلے کمانڈز کی دستیابی چیک کریں؛ تجویز کرنے سے پہلے جانچ کریں

### Git کے اصول

- کبھی بھی commit hash ایجاد نہ کریں — `git log` استعمال کریں
- کبھی بھی commit پیغامات کا قیاس نہ کریں — `git log --oneline` استعمال کریں
- کبھی بھی PR نمبر یا Issue IDs ایجاد نہ کریں
- commit کے بعد، `git status` اور `git log -1` چلائیں
- برانچ کے وجود کا دعویٰ کرنے سے پہلے `git branch -a` چلائیں

### اعتماد کی سطحیں

- **تصدیق شدہ**: میں نے براہ راست پڑھا/چلایا/جانچا
- **امکان ہے**: مضبوط ثبوت لیکن براہ راست تصدیق نہیں ہوئی
- **قیاسی**: تخمینہ، تصدیق کی ضرورت ہے
- **نامعلوم**: دستیاب معلومات سے تعین نہیں کیا جا سکتا

طے شدہ طور پر **تصدیق شدہ** استعمال کریں — ہمیشہ اس سطح تک پہنچنے کے لیے ٹولز استعمال کریں۔

### شدت بڑھانے کا پروٹوکول — رکیں اور پوچھیں جب:

1. ہدایات مبہم ہوں اور متعدد تشریحات ممکن ہوں
2. مطلوبہ ٹول/لائبریری دستیاب نہ ہو
3. آپریشن کو بلند اجازتوں کی ضرورت ہو
4. آپ کا مشاہدہ صارف کی وضاحت سے متصادم ہو
5. کام میں ایسی معلومات درکار ہوں جن کی آپ تصدیق نہیں کر سکتے
6. آپریشن ڈیٹا کے نقصان یا سیکورٹی کے مسائل کا سبب بن سکتا ہو
7. سیاق و سباق میں درست طریقے سے ٹریک کرنے کے لیے بہت زیادہ تبدیلی آئی ہو
